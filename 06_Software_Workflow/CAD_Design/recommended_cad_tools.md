# Recommended CAD Tools

---

## Overview

The GT2 accepts STL files as input to DeScribe. Any software capable of producing a clean, watertight STL file is compatible with the GT2 workflow. There is no official Nanoscribe-endorsed CAD tool — the choice depends on what you are designing, your familiarity with different tools, and what your institution provides.

This file describes the most commonly used tools in GT2 labs and their specific strengths and limitations for nanofabrication design.

---

## SolidWorks

Best for: precise parametric solid modeling of mechanical and microoptical structures.

SolidWorks is the most widely used professional CAD tool in engineering research labs. It models solid bodies using features (extrude, revolve, sweep, loft) driven by sketches with exact dimensions. Every dimension is parametrically defined and can be changed at any time without redrawing the model.

Strengths for GT2 use:
- Extremely precise dimension control — specify features to nanometer precision if needed
- Parametric modeling makes it easy to create design variants (scale one parameter and everything updates)
- Built-in STL export with mesh quality control
- Excellent for rotationally symmetric structures (lenses, pillars, rings)
- Assembly mode allows designing structures made of multiple components before combining into a single print volume

STL export from SolidWorks:
File > Save As > STL > Options: set resolution to Fine or Custom (deviation 0.001 mm, angle 0.5 degrees) > Save

Limitations:
- Expensive license — typically requires institutional access
- Steeper learning curve than free alternatives
- Organic or freeform shapes require additional surfacing features

## Autodesk Fusion 360

Best for: general 3D design including mechanical structures, microfluidics, and freeform surfaces.

Fusion 360 is a cloud-based CAD/CAM tool that combines parametric solid modeling with mesh and surface modeling. It is available free for students, educators, and startups.

Strengths for GT2 use:
- Free for academic use
- Combines parametric and freeform modeling in one environment
- Good STL export control
- Widely used — extensive tutorial resources available
- Generative design tools for topology-optimized lattice structures (useful for 10x large-volume prints)

STL export from Fusion 360:
File > Export > STL > Refinement: High > OK. Alternatively, right-click the body in the browser > Save as Mesh > STL > High refinement.

Limitations:
- Requires an internet connection for cloud save (local save available with paid plans)
- Some complex surface operations are less stable than SolidWorks
- Mesh-to-STL conversion of organic shapes sometimes produces non-manifold geometry that requires repair

## OpenSCAD

Best for: programmatically defined parametric structures, arrays, and structures generated by mathematical functions.

OpenSCAD is a free, open-source script-based CAD tool. Instead of drawing a model graphically, you write a program that describes the geometry. This makes it ideal for structures defined by equations — photonic crystals, Schwartz surfaces, gyroid lattices, periodic arrays.

Strengths for GT2 use:
- Completely free and open source
- Parametric by design — every dimension is a variable in code
- Excellent for periodic structures and mathematical surfaces
- Runs on all operating systems
- Models are version-controllable (text files, compatible with Git)

Example OpenSCAD code for a simple pillar:

    diameter = 5;   // in um
    height = 20;    // in um
    cylinder(h = height, d = diameter, center = false, $fn = 100);

STL export from OpenSCAD:
Design > Export as STL. For curved surfaces, increase $fn (number of facets) to 100 or higher before export.

Limitations:
- No graphical modeling interface — requires comfort with code
- Complex freeform shapes are difficult to express programmatically
- Preview rendering can be slow for complex geometry
- Error messages are sometimes cryptic

## FreeCAD

Best for: free alternative to SolidWorks for parametric solid modeling.

FreeCAD is a free, open-source parametric CAD tool that follows a similar workflow to SolidWorks. It is less polished but fully functional for most GT2 design tasks.

Strengths:
- Completely free
- Parametric solid modeling
- Active development community
- Python scripting interface for automation

STL export from FreeCAD:
File > Export > STL Mesh. Use Mesh Workbench for mesh quality settings: Meshes > Create Mesh from Shape > Maximum edge length 0.001 mm.

Limitations:
- Less stable than commercial alternatives — occasional crashes on complex operations
- UI is less intuitive than Fusion 360 or SolidWorks
- Some advanced features require switching between different workbenches

## Blender

Best for: organic shapes, artistic or biological structures, and complex freeform geometry.

Blender is a free, open-source 3D modeling tool primarily used for animation and visualization but fully capable of producing printable geometry.

Strengths for GT2 use:
- Exceptional freeform and organic modeling capability
- Free and open source
- Python scripting for algorithmic geometry generation
- Good for biological structures (cell-like shapes, anatomically accurate scaffolds)

Critical limitation for GT2 use:
Blender works with meshes, not solid geometry. Mesh models frequently have non-manifold geometry, duplicate vertices, or open boundaries that cause problems in DeScribe. Always run the STL through a repair tool (Meshmixer, Netfabb, or Microsoft 3D Builder) before importing into DeScribe.

STL export from Blender:
File > Export > STL. Apply all modifiers before export. Run the model through Mesh > Cleanup > Fill Holes and Mesh > Normals > Recalculate Outside before exporting.

## MATLAB and Python (Programmatic STL Generation)

For researchers who need to generate structures from mathematical expressions, analytical functions, or computational design algorithms, it is possible to write STL files directly from MATLAB or Python without using CAD software at all.

Python libraries for STL generation:
- numpy-stl: create STL files from numpy arrays of triangle vertices
- trimesh: powerful mesh processing, repair, and STL export
- PyMesh: advanced mesh operations

MATLAB:
- Patch objects can be exported as STL using custom scripts or the stlwrite function

This approach is the most powerful for research applications — you can define a structure as a mathematical function and directly generate the STL without any graphical interface.

## STL Repair Tools

Regardless of which CAD tool you use, always validate your STL before importing into DeScribe. Useful repair tools:

Microsoft 3D Builder (Windows, free):
Open the STL file. If any issues are found, 3D Builder prompts you to repair automatically. Reliable for simple repairs.

Meshmixer (Autodesk, free):
Analysis > Inspector shows all mesh errors as red pins. Click Auto Repair to fix most issues. Good for more complex problems.

Netfabb (Autodesk):
The industry standard for STL repair. Free version available at netfabb.autodesk.com. Upload the file and it returns a repaired version.

PrusaSlicer (free):
Although designed as a slicer for FDM printers, PrusaSlicer includes good STL validation. If PrusaSlicer can slice your model without warnings, it is likely watertight.
